About This Project

An interactive 3D product showcase using Three.js and modern web technologies

Project Overview

This website was developed as part of a web development course focused on integrating 3D models into websites. The goal was to create an interactive experience that allows users to view and interact with 3D product models directly in their browser.

The website demonstrates the integration of Three.js for 3D rendering, along with animations, sound effects, and user interactions to create an engaging product showcase experience.

Core Features

3D Model Integration

The website loads and displays GLB format 3D models with proper lighting, camera setup, and scene management. Models include animated elements that respond to user interactions.

Interactive Controls

Users can interact with the 3D models through buttons that trigger animations and transformations. The camera system allows orbiting and zooming for a complete view of each product.

Audio Integration

Sound effects are triggered during animations, enhancing the immersive experience when interacting with products like opening a Pepsi can.

Responsive Design

The website is fully responsive, adapting to different screen sizes while maintaining the functionality of the 3D viewer across devices.

Technical Implementation

3D Rendering with Three.js

The project uses Three.js for WebGL-based 3D rendering, with custom camera positioning and lighting setups tailored to each product model. The rendering pipeline includes:

  • Scene and camera initialization with proper aspect ratios
  • Dynamic lighting setup with ambient and directional lights
  • Model loading with GLTFLoader and animation mixing
  • Interactive OrbitControls for user viewing angle adjustment

Animation System

Animations are handled through Three.js AnimationMixer to control playback of model animations. Each product has custom animation sequences that can be triggered by user interactions.

3D Model Preparation in Blender

The 3D models were created and animated in Blender, then exported to GLB format for web compatibility. I encountered significant issues during export where the textures and materials applied in Blender did not transfer correctly to the GLB format. The models looked completely different after export due to these texture and material restrictions, requiring additional adjustments to achieve the desired appearance.

Event-Driven Interactions

The application uses event listeners to respond to user actions, such as clicking buttons to trigger animations, toggling wireframe mode, or rotating models.

"The most challenging aspect was synchronizing animations with sound effects to create a realistic experience while maintaining performance."

Development Process

Planning and Design

The development process began with sketching the website layout and planning the user interactions. The design focused on creating an intuitive interface that highlights the 3D models while providing clear controls.

3D Model Preparation

The 3D models were optimized for web delivery, with careful attention to polygon count, texture sizes, and animation complexity to ensure good performance across devices.

Implementation Challenges

Key challenges included handling different model types with a unified codebase, ensuring animations played correctly, and managing the audio integration across browsers.

3D Format Export Challenges

I was unable to export my model with the textures that I applied in the Blender file. The model looked completely different due to the texture or material restrictions while exporting the .glb file. This was a significant issue that required additional work to resolve, as the visual appearance in Blender didn't translate properly to the web-compatible format.

Testing and Refinement

The website underwent extensive testing across different browsers and devices to ensure consistent functionality and performance, with special attention to mobile optimization.

Future Enhancements

Future plans for this project include:

  • Adding more product models and animations to the showcase
  • Implementing AR (Augmented Reality) viewing options for mobile devices
  • Creating interactive product customization features
  • Improving performance optimizations for slower devices
